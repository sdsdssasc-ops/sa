<a id="2"></a>
## الخطوة 1: إنشاء حساب GitHub ورفع الملفات

GitHub هو "مخزن" ملفات المشروع، وRender بيسحب الملفات منه.

### أ) إنشاء الحساب

1. افتح **https://github.com**
2. اضغط **Sign up**
3. أدخل بريدك، كلمة مرور، واسم مستخدم (أي اسم متاح)
4. فعّل حسابك من البريد اللي يوصلك

### ب) إنشاء مستودع (Repository) جديد

1. بعد تسجيل الدخول، اضغط علامة **+** أعلى يمين الصفحة ← **New repository**
2. في خانة **Repository name** اكتب: `taafy-app` (أو أي اسم يعجبك)
3. خلّه **Public**
4. **لا** تعلّم على أي خيار من الخيارات تحت (لا README ولا .gitignore — المشروع فيها كل شي)
5. اضغط **Create repository**

### ج) رفع ملفات المشروع

**الطريقة الأسهل (من المتصفح — بدون برامج):**

1. فكّ ضغط ملف ZIP على جهازك، بيطلع لك مجلد فيه ملفات المشروع
2. في صفحة المستودع الفارغة بـ GitHub، اضغط رابط **uploading an existing file**
3. **اسحب كل ملفات ومجلدات المشروع** (اللي داخل مجلد `app` — مو مجلد `app` نفسه) إلى الصفحة
   - يعني تسحب: `src` و `server` و `public` و `package.json` و `render.yaml` وباقي الملفات
   - تأكد أنك تسحب **المحتويات**، بحيث يكون ملف `render.yaml` في الصفحة الأولى للمستودع (مو داخل مجلد فرعي)
   - **ملاحظة:** إذا قال GitHub إن الملفات كثيرة، ارفعها على دفعتين أو ثلاث
4. انزل تحت واضغط **Commit changes**

**طريقة الأوامر (إذا تعرف تستخدم الطرفية):**

```bash
cd مسار/مجلد/المشروع
git init
git add .
git commit -m "موقع تعافي - النسخة الأولى"
git branch -M main
git remote add origin https://github.com/اسم_المستخدم/taafy-app.git
git push -u origin main
```

### د) تأكد من الرفع

افتح صفحة المستودع، لازم تشوف في الصفحة الأولى مباشرة:
- مجلد `src` ✅
- مجلد `server` ✅
- مجلد `public` ✅
- ملف `render.yaml` ✅ ← **هذا أهم ملف للنشر التلقائي**
- ملف `package.json` ✅

> **ما لازم تشوف:** مجلد `node_modules` أو `dist` أو ملف `.env` — هذه محظورة في `.gitignore` (كبيرة أو سرية).

<a id="4"></a>
## الخطوة 3: النشر بضغطة واحدة (Blueprint)

هنا السحر — ملف `render.yaml` اللي جهزته لك يسوي كل شي تلقائيًا: **الموقع + قاعدة البيانات + ربطهم ببعض**.

1. من لوحة Render (Dashboard) اضغط **New +** ← **Blueprint**
2. بيطلب صلاحية الوصول لمستودعاتك: اضغط **Connect GitHub** ← اختر **All repositories** أو اختر مستودع `taafy-app` فقط ← **Install**
3. اختر مستودع `taafy-app` من القائمة واضغط **Connect**
4. بيكتشف ملف `render.yaml` ويعرض لك ملخص: خدمة ويب `taafy-app` + قاعدة بيانات `taafy-db` — الاثنين **Free**
5. اضغط **Apply** / **Deploy Blueprint**
6. انتظر 3–5 دقائق: Render يثبّت المكتبات، يبني الواجهة، ينشئ قاعدة البيانات، ويشغّل كل شي
7. لما تتحول الحالة إلى **Live** (أخضر) ← اضغط على الرابط أعلى الصفحة (شكله: `https://taafy-app.onrender.com`)

**مبروك! موقعك على الإنترنت.** 🎉

> أول فتحة ممكن تأخذ 30–50 ثانية إذا الموقع كان نايم (طبيعي في الخطة المجانية — شرحها تحت).

<a id="5"></a>
## الخطوة 4: أول دخول وتأمين حساب الأدمن

1. افتح رابط موقعك ← **تسجيل الدخول**
2. ادخل ببيانات الأدمن الافتراضية:
   - البريد: `admin@taafy.local`
   - كلمة المرور: `admin123456`
3. بيدخلك مباشرة على **لوحة الأدمن** — جرّبها

### غيّر بيانات الأدمن فورًا (مهم!)

1. في Render افتح خدمة **`taafy-app`**
2. من القائمة اليسار اختر **Environment**
3. عدّل:
   - `ADMIN_EMAIL` ← حط بريدك الحقيقي
   - `ADMIN_PASSWORD` ← حط كلمة مرور قوية
4. اضغط **Save Changes** — Render يعيد تشغيل الموقع تلقائيًا (دقيقة تقريبًا)
5. سجّل دخولك بالبيانات الجديدة

> ملاحظة: لو سجّلت كمستخدم عادي بنفس بريد الأدمن قبل ما تغيّره، النظام يرقّيه تلقائيًا لأدمن عند إعادة التشغيل.

<a id="6"></a>
## إضافة الفيديو التعريفي

عندك طريقتين — اختر اللي يناسبك:

### الطريقة 1: رفع ملف الفيديو مع الموقع

1. سمّ ملف الفيديو بالضبط: **`intro.mp4`**
2. حطه في مجلد **`public/videos/`** داخل المشروع
3. ارفعه على GitHub (نفس طريقة الرفع في الخطوة 1-ج)
4. Render يعيد النشر تلقائيًا والفيديو يظهر في الصفحة الرئيسية

⚠️ **حدود مهمة:** GitHub ما يقبل ملف واحد أكبر من **100 ميجا**، والأفضل عمليًا ما يتجاوز **30–50 ميجا** عشان ما يثقل الموقع. إذا فيديوك أكبر، استخدم الطريقة 2.

### الطريقة 2: فيديو يوتيوب (الأفضل للفيديوهات الكبيرة)

1. ارفع الفيديو على قناتك في يوتيوب (تقدر تخليه "غير مدرج/Unlisted" إذا تبيه خاص بزوار الموقع فقط)
2. خذ معرّف الفيديو من الرابط:
   - رابطك: `https://www.youtube.com/watch?v=`**`dQw4w9WgXcQ`**
   - المعرّف هو الجزء اللي بعد `v=` ← `dQw4w9WgXcQ`
3. افتح ملف **`src/config.ts`** في GitHub (اضغط على الملف ← أيقونة القلم ✏️)
4. في سطر `youtubeId: ''` حط المعرّف بين علامتي التنصيص: `youtubeId: 'dQw4w9WgXcQ'`
5. اضغط **Commit changes** — ينعكس تلقائيًا خلال دقائق

> إذا ما فيه فيديو أصلًا، الموقع يعرض بطاقة أنيقة مكانه — ما يخرب الشكل أبدًا.

<a id="7"></a>
## تخصيص الموقع (الاسم، الأسعار، المختصون…)

**كل المحتوى القابل للتعديل مجمّع في ملف واحد:** `src/config.ts`

عدّله مباشرة من GitHub (افتح الملف ← أيقونة القلم ✏️ ← Commit):

| تبي تغيّر | عدّل |
|---|---|
| اسم الموقع ووصفه | `site.name` و `site.description` |
| البريد / الجوال / الواتساب / المدينة | `site.email` و `site.phone` و `site.whatsapp` و `site.city` |
| أنواع الجلسات وأسعارها ومددها | قائمة `sessionTypes` |
| الأوقات المتاحة للحجز | قائمة `timeSlots` |
| أسماء المختصين وسيرهم | قائمة `specialists` |

**مثال** — تغيير سعر جلسة التأمل:
```ts
{
  id: 'جلسة تأمل موجّه',
  name: 'جلسة تأمل موجّه',
  duration: '٣٠ دقيقة',
  price: '٩٠ ر.س',        // ← غيّرت السعر هنا
  description: 'تأمل صوتي موجّه...',
},
```

الأسعار والأوقات تظهر تلقائيًا في الصفحة الرئيسية **وفي نموذج الحجز** — ما تحتاج تعدّل مكانين.

> بعد أي Commit في GitHub، Render يعيد بناء الموقع ونشره تلقائيًا خلال 2–4 دقائق (ميزة Auto-Deploy مفعّلة افتراضيًا).

<a id="8"></a>
## ملاحظات مهمة عن الخطة المجانية

### أ) الموقع "ينام" بعد 15 دقيقة بدون زيارات

- أول زائر بعد النوم ينتظر **30–60 ثانية** ويفتح الموقع، بعدها يرجع سريعًا للجميع
- هذا طبيعي ومو عطل — هكذا الخطة المجانية في Render
- لو أزعجك لاحقًا: الترقية للخطة المدفوعة (7$ شهريًا) تخليه صاحي 24/7، أو تستخدم خدمة ping مجانية مثل **UptimeRobot** توقظه كل 10 دقائق (حل شائع ومجاني)

### ب) قاعدة بيانات Render المجانية تنتهي بعد 30 يوم

- Render يعطيك PostgreSQL مجاني لمدة 30 يوم، بعدها يحذفه (وينبّهك بالبريد قبلها)
- **عندك حلول:**
  1. **الأسهل:** جدّد بقاعدة مجانية جديدة من Render (تسوي Delete للقديمة وتنشئ وحدة جديدة وتربطها) — بس البيانات القديمة تروح
  2. **الأفضل للمدى الطويل (مجاني دائم):** استخدم **Neon** — قاعدة PostgreSQL مجانية بدون انتهاء:
     - سجّل في **https://neon.tech** (بحساب GitHub)
     - Create project ← انسخ رابط الاتصال (Connection string)
     - في Render ← خدمة `taafy-app` ← **Environment** ← عدّل قيمة `DATABASE_URL` وحط رابط Neon ← Save
     - خلاص — موقعك يشتغل على قاعدة دائمة مجانية

### ج) رابط الموقع

- الرابط المجاني: `https://taafy-app.onrender.com` (اسم الخدمة تحدد الرابط)
- تقدر تربط دومين خاص فيك (مثل `taafy.com`) لاحقًا من Settings ← Custom Domains — الربط مجاني، بس الدومين نفسه تشتريه من أي مسجّل

<a id="9"></a>
## كيف تحدّث الموقع بعد النشر

أي تعديل على GitHub = تحديث تلقائي للموقع:

1. افتح المستودع في GitHub
2. افتح الملف اللي تبي تعدّله ← أيقونة القلم ✏️
3. عدّل ← **Commit changes**
4. Render يعيد البناء والنشر تلقائيًا (تابع التقدم من Render ← Events)

> تبي تعطّل النشر التلقائي؟ Render ← Settings ← Build & Deploy ← Auto-Deploy: Off

<a id="10"></a>
## حل المشاكل الشائعة

| المشكلة | السبب | الحل |
|---|---|---|
| **Deploy failed** أثناء البناء | غالبًا `render.yaml` مو في الصفحة الأولى للمستودع | تأكد أن الملفات مرفوعة مباشرة (مو داخل مجلد فرعي) |
| الموقع يفتح بس تسجيل الدخول يطلّع خطأ | مشكلة اتصال قاعدة البيانات | افتح Render ← خدمة `taafy-app` ← **Logs** وشوف الأخطاء. تأكد أن `taafy-db` حالتها Available |
| أول فتحة بطيئة جدًا | الموقع كان نايم | طبيعي — انتظر دقيقة وحدّث الصفحة |
| نسيت كلمة مرور الأدمن | — | Render ← Environment ← غيّر `ADMIN_PASSWORD` ← Save ← ادخل بالجديدة |
| الفيديو ما يظهر | اسم الملف غلط أو مكانه غلط | لازم يكون `public/videos/intro.mp4` بالضبط، أو استخدم `youtubeId` |
| الرابط يطلّع Not Found | البناء لسا ما خلص | انتظر حتى تتحول الحالة Live |
| نسيت بريد مستخدم مسجّل | — | من لوحة الأدمن ← تبويب "المستخدمون" تشوف كل المسجّلين |

### كيف تشوف سجل الأخطاء (Logs)

Render ← اضغط `taafy-app` ← **Logs** من القائمة اليسار — كل شي يتسجّل هنا بالعربي تقريبًا (رسائل السيرفر كلها عربية).

## التشغيل على جهازك (اختياري — للتطوير)

إذا بتعدّل الكود بنفسك:

1. ثبّت **Node.js 20+** من https://nodejs.org
2. ثبّت **PostgreSQL** من https://www.postgresql.org/download (أو استخدم رابط Neon المجاني بدون تثبيت شي)
3. في مجلد المشروع:
```bash
# انسخ ملف البيئة وعبّيه
cp .env.example .env        # ثم افتح .env وعدّل DATABASE_URL

npm install                 # تثبيت المكتبات (أول مرة فقط)
npm run dev:server          # السيرفر الخلفي على http://localhost:3001
npm run dev                 # (في نافذة ثانية) الواجهة على http://localhost:3000
```
4. افتح http://localhost:3000 — أي تعديل بالكود ينعكس فورًا

**قبل رفع تعديلاتك على GitHub تأكد أن البناء يشتغل:**
```bash
npm run build               # لازم يخلص بدون أخطاء
```
